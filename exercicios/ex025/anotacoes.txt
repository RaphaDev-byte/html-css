
o form precisa de um input para receber os dados.

o form precisa ter um action para ele ser enviado para o servidor.

identificar os inputs é muito importante para as linguagens. name é mais util para html e php e id mais util para javascript.

nao se faz formulario sem relacionar o label ao seu elemento pois dificulta o navegador de identificar.

o label é muito importante.

quando o formulario é enviado a url fica da seguinte forma:

/exercicios/ex025/cadastro.php?nome=batata&sobrenome=raiz

a ? divide oque é a url e os parametros extras que foram enviados.

existem dois metodos para envio de formulario.
por padrao o form é get.
mas tambem tem o post.

no caso do post os parametros nao ficam na cara do usuario mas é possivel ver atraves do inspecionar.

para realmente tornar seguro e proteger os dados é preciso criptografar se aprofundar em segurança.

get é util para quando os dados nao sao sensiveis. quando usar o get so pode enviar ate 3000 bytes, oque é aproximadamente 3000 letras.

https é necessario para proteger dados. estudar para reforcar seguranca.

atributos:
required torna o campo obrigatorio a ser preenchido.

minlenght faz com que o campo tenha um minimo de letras. 

maxlenght faz com que tenha no max x de letras.

size faz a caixa ficar de determinado tamanho.

placeholder faz aparecer um texto dentro da caixa.

autocomplete off ou on no form desliga e liga as sugestoes. autocomplete tambem fica dentro dos inputs para o navegador saber com oque esta lidando para dar sugestao seja de senha, usuario e outros.


o type = number so aceita numeros e tem os atributos min, max que definem os numeros min e max ex:0 a 10 e step decide o espacamento entre os valores. por exemplo 0.5

vericar se o navegador é compativel com os atributos do html.

estudar expressoes regurares, reGex.

pattern cria padroes a serem seguidos por exemplo para nao faltar um numero no telefone.

a tag FIELSET agrupa os campos e legend coloca uma legenda, olhar form004.

inputs radio com o mesmo name so permitem que um deles sejam selecionados por exemplo masculino ou feminino, olhar o form005, colocar checked como atributo no input ja deixa marcado.

quando usar o controle de envio de arquivo so pode usar o metodo POST.


select é outro tipo de campo onde voce seleciona, dentro existem os options onde podem ser agrupados com o optgroup.
