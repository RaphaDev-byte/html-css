Media Queries. 

para diferentes telas ou formatos é preciso configuracoes de estilo para melhor desingn e aproveitamento do conteudo, pra isso servem as medias queries.

é bom criar arquivos css para cada formato, por exemplo um para tela de computador, outra pra celular e impressao e outros.

exemplo:

<link rel="stylesheet" href="estilos/style.css" media="screen">
    <link rel="stylesheet" href="estilos/impressora.css" media="print">

    aqui tem um estilo para a tela e outro para impressao.

    Media Query é igual a Media Type mais Media Feature





    orientaçao, landscape(aparelho deitado, largura maior que a altura) e portrait(aparelho em pé, altura maior que a largura)


    quando eu uso o media eu posso informar medias types e medias features.
    
    ex:
    <link rel="stylesheet" href="retrato.css" media="screen and (orientation:portrait)">
    <link rel="stylesheet" href="estilos/paisagem.css" media="screen and (orientation:landscape)">



    é possivel unificar todos os estilos na propria pagina do html, mas nao é recomendavel para projetos grandes ja que fica um pouco baguncado. olhar o index.html da pasta mq003 



    First mobile.

    ha muito tempo, era normal primeira ser feito o site focado no desktop e depois de finalizado cortavam coisas e retiravam coisas para a versao mobile, oque resultava em uma experiencia nao tao boa para o usuario mobile, ate que em 2009 um cara que trabalhava na Yahoo criou a ideia de faazer primeiro focado pro mobile e depois ia pro desktop, e no fim essa era e é um otimo jeito pelos principais motivos:

    performace: site focado primeiro em ser leve e eficiente, dando uma melhor acessibilidade para quem nao tem interet mais precaria.

    melhor experiencia do usuario e menos trabalho: nao precisavam cortar ou arrancar partes do codigo, era so implementar coisas a mais para o desktop e pequenas alteracoes.

    os mecanismos do google dao mais atençao para sites que tem essas caracteristicas acima. Sò vantagem!


    criaçao do mobile First no mq004, olhar o tela.css para informacoes.
